# AGENTS.md — 自動化エージェント規律

本書は LLM / Jules / 自動生成パイプラインが**機械的に遵守すべき規律**を定める。
仕様の全体像は [SPEC.md](./SPEC.md) を参照。**コンプラ §2 は最優先・例外なし。**

## 1. プロジェクト規律

- **TODO・課題は GitHub Issues (omochairo/invest-content-studio) で一次管理**。memory やコード内 TODO に長期課題を書かない（omochairo で stale 化して混乱した実績）。
- 作業は1タスクずつ。結果を確認してから次へ。
- 1回の書き込みで150行を超えるファイルを作らない。長い文書は分割する。
- ブランチを切ってから作業。`main` への直 push は避け、PR 経由にする。

## 2. コンプラ — 金融商品取引法（最優先・例外なし）

個別銘柄の売買推奨や断定的判断の提供を反復継続すると**投資助言業（登録制）**に該当し、
無登録は違法。**生成段階でブロックする。** 出力は常に「教育・情報提供・事実報道・一般論」に限定する。

### 2.1 禁止表現（コンプラ gate がブロック）

以下の意味を持つ表現を**生成しない**。検出したらその成果物を fail させる。

- **売買の推奨・指示**：「買うべき」「今が買い時」「売り時」「仕込め」「狙い目」「買い推奨」
- **断定的な将来予測**：「必ず上がる/下がる」「確実に」「間違いなく」「絶対」「〜は確定」
- **利回り・元本の保証**：「儲かる」「損しない」「元本保証」「リスクなし」「年利◯%確実」
- **個別銘柄への投資判断の名指し誘導**：「○○（銘柄）は今すぐ買い」等

### 2.2 必須要素（コンプラ gate が存在を検証）

- **免責文**：全成果物に `meta.disclaimer` を自動挿入（「本コンテンツは情報提供を目的としたもので、
  特定の金融商品の売買を推奨・勧誘するものではありません。投資判断はご自身の責任で行ってください。」相当）。
- **出典**：`meta.sources` に一次情報リンクを最低1件。空なら fail。
- **PR表記**：アフィリエイト導線がある成果物には PR / 広告表記を自動付与（景表法・ステマ規制）。

### 2.3 表現の置き換え方針

- ×「この銘柄は買い」→ ○「○○社は決算を開示した。要点は△△（出典）」＝**事実報道に倒す**
- ×「必ず上がる」→ ○「市場ではこう報じられている／こういうデータがある」＝**主語を事実・出典に移す**

## 3. 著作権 / 情報源

- **一次情報起点が原則**：SEC EDGAR filing / TDnet 適時開示 / 各社IR / 価格API（CoinGecko 等）/ 公的統計。
- 他社記事（日経・Reuters・Bloomberg 等）の**転載・要約転載を禁止**。事実（数値・開示の有無）のみを
  一次情報から取得し、出典を明示する。
- 引用が必要な場合も主従関係・出典明示・必要最小限を守る。

## 4. ContentPackage 契約の遵守

- 生成エンジンの出力は [SPEC.md §3](./SPEC.md) の `ContentPackage` スキーマに**必ず適合**させる。
  スキーマ違反は validate で fail。
- ドメイン固有のロジックを**レンダラー（Remotion）や TTS 層に漏らさない**。投資もおもちゃも
  同じ ContentPackage を吐けば同じレンダラーが動く＝逆輸入の前提を壊さない。
- TTS / 動画エンジンの差し替えは ContentPackage 契約の裏で行う。契約を変えるときは SPEC.md を先に更新。

### 4.1 台本生成の二段構え（Gemini / Jules）

JP 長尺の台本生成は **prose（title / narration / caption）のみ** を LLM に委ね、**数値・チャート・stat は
すべてコード由来固定**（§3 非ハルシ契約）。これは生成エンジンに依らず不変。

- **日次・量産 = Gemini**（`generate:jp`）。同期・安価。`gemini-2.5-flash`→`-lite` 日次 quota フォールバックあり。
- **選抜・深掘り = Jules**（`request:jules:jp` → AUTO_CREATE_PR で `jules-research/<code>.prose.json` を追加
  → マージ後 `harvest:jules:jp` / `phase2-jp.yml gen_source=jules`）。Jules は非同期エージェントで時間をかけ
  深く調査するが、**書いてよいのは prose だけ**。harvest は Gemini 経路と**同一の assemble + validate +
  §2 gate** を通すので、Jules prose も §2 違反は確実にブロックされる（`verify:jules` が回帰検証）。
- **Jules quota は omochairo と共有**（Pro 100/日・rolling 24h）。`request:jules:jp` は直近作成数で gate し、
  予算切れ時は no-op。**自動 schedule では回さず手動 dispatch のみ**（omochairo の記事生成を枯渇させない）。

## 5. 自動化ワークフロー規律

- 生成タスクは **fetch → ContentPackage 生成 → validate（スキーマ）→ コンプラ gate → レンダリング →
  commit → PR** を一連で自走する。各ステップ手前で「進めていいか」と停止しない（omochairo の
  AWAITING_USER_FEEDBACK 浪費を踏襲しない）。
- ただし**コンプラ gate fail は自走を止める**。修正せず公開へ進めることは禁止。
- 配信は当初**限定公開**で出力し、レビュー後に公開へ昇格する。

## 6. 品質ゲートの順序（要点）

1. **validate** — ContentPackage がスキーマ適合か
2. **コンプラ gate** — §2 の禁止表現ゼロ・必須要素あり（最重要）
3. **レンダリング** — Remotion で mp4 化、字幕・音声同期の健全性
4. fail はいずれも自走停止せず修正ループへ。ただし §2 fail のみ公開不可で確実に止める。
